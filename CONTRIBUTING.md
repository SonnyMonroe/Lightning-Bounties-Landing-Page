# Contributing to Lightning Bounties

Thank you for your interest in contributing to Lightning Bounties! We are an open-source project and welcome contributions from the community.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue on our [GitHub Repository](https://github.com/Lightning-Bounties/landing-page/issues) with the following details:
- Description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Screenshots (if applicable)

### 2. Suggesting Enhancements
Have an idea to make Lightning Bounties better? We'd love to hear it! Open an issue with the "enhancement" label and describe your idea in detail.

### 3. Pull Requests
We actively welcome your pull requests.

1.  **Fork** the repo on GitHub.
2.  **Clone** the project to your own machine.
3.  **Commit** changes to your own branch.
4.  **Push** your work back up to your fork.
5.  Submit a **Pull Request** so that we can review your changes.

NOTE: Be sure to merge the latest from "upstream" before making a pull request!

## Development Setup

1.  Install Node.js and npm.
2.  Run `npm install` to install dependencies.
3.  Run `npm run dev` to start the local development server.

## Code Style
- We use **TypeScript** for type safety.
- We use **Tailwind CSS** for styling.
- Please ensure your code is formatted cleanly (Prettier is recommended).

## Community
